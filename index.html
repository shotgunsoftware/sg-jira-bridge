The Flow Production Tracking Jira Bridge (sg-jira-bridge) provides a simple setup to sync data between Flow Production Tracking and Jira.

At the heart of the workflow is a lightweight webapp that handles POST requests to sync data. A request is validated and executed by this module code which handles all of the connections, logic, and data translation between Flow Production Tracking and Jira. It then sends a request to update Flow Production Tracking or Jira with the translated data.

Requests to sync data from Flow Production Tracking to Jira are generated by the sg_jira_event_trigger.py plugin running in the Flow Production Tracking Event Daemon. Requests to sync data from Jira to Flow Production Tracking are generated by a web hook that is configured in Jira for your Project.
